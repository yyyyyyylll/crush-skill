# 修正模式 — Correction Handler

## 目标

当用户反馈"TA 实际回了 X"或"你预测错了"时，对比预测与实际，找出画像中的偏差并更新。

---

## 触发条件

用户可能的说法：
- "TA 实际回了..."
- "你猜错了，TA 说的是..."
- "预测不准，TA..."
- "TA 没回" / "TA 已读不回了"
- "TA 的反应完全不一样"

## 执行流程

### Step 1: 收集对比数据

向用户确认：
1. **你发了什么消息？**（如果之前预测过，直接引用）
2. **TA 实际怎么回的？**（原文或描述）
3. **回复时间大概多久？**

### Step 2: 差异分析

将预测与实际在以下维度对比：

| 维度 | 预测 | 实际 | 偏差 |
|------|------|------|------|
| 情感倾向 | 积极/中性/消极 | ? | |
| 热情度 | N/10 | ? | |
| 回复时间 | 范围 | ? | |
| 回复长度 | 长/中/短 | ? | |
| 是否匹配预测变体 | A/B/C/不回 | ? | |

### Step 3: 根因追溯

确定哪个画像层导致了预测偏差：

- **Layer 2 偏差**（沟通风格）：回复速度、长度、emoji 用法与预期不同
  → 更新 Layer 2 对应字段
  
- **Layer 3 偏差**（情绪模式）：对特定话题的反应与预期不同
  → 更新 Layer 3 对应字段

- **Layer 4 偏差**（关系动态）：互动热度与预期不同
  → 更新 Layer 4，可能需要重新计算温度

- **消息类型判断偏差**：预测引擎对消息类型归类错误
  → 记录但不更新画像

- **外部因素**：TA 可能当时心情不好/特别忙/在特殊场景
  → 记录为异常值，不立即更新画像

### Step 4: 更新画像

```
更新规则:
1. 单次偏差: 记录到 corrections 数组，暂不改置信标签
2. 同方向偏差 ≥2 次: 调整字段值，置信标签升级
3. 矛盾偏差（有时准有时不准）: 增加字段的 variance 描述
```

调用 `tools/profile_manager.py update` 保存更新后的画像。

### Step 5: 反馈给用户

---

## 输出格式

```
═══ 预测修正 ═══

📋 对比:
  预测: {预测内容摘要}
  实际: {实际内容}

🔍 偏差分析:
  {情感倾向}: 预测{X} → 实际{Y}
  {热情度}: 预测{N}/10 → 实际约{M}/10
  {回复时间}: 预测{范围} → 实际{时间}

💡 学到了什么:
  {具体发现，例如：TA 对邀约类消息比预期更积极}
  
🔄 画像更新:
  • {Layer}.{field}: {旧值} → {新值} [置信度: {标签}]
  
📊 累计预测准确率: {correct}/{total} ({pct}%)

{如果准确率 < 50% 且 ≥5 次预测}:
⚠️ 预测准确率较低，建议补充更多聊天数据来提高画像精度。
```

---

## 正面反馈处理

如果用户说"你猜对了"或实际回复与预测匹配：

```
═══ 预测命中 ✓ ═══

📋 预测匹配变体: {A/B/C}
📊 累计准确率: {correct}/{total} ({pct}%)
💡 这说明画像中 {相关字段} 的判断是准确的。
```

同样记录到 corrections 数组（作为正面数据点）。

---

## 注意事项

- 不要为预测不准道歉——重点在"学到了什么"和"怎么改进"
- 不要过度拟合单次偏差。一次意外回复可能是特殊情况，不一定需要重大调整
- 如果用户情绪低落（因为收到了冷回复），先共情再分析
- 鼓励用户持续反馈，因为每次修正都在让预测更准
