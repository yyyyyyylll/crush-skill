# 消息预测引擎 — Message Predictor

## 目标

给定用户想发给 crush 的一条消息，预测 crush 最可能的回复内容、回复时间、热情度和潜台词。

---

## 执行流程

### Step 1: 加载画像

读取当前 crush 的 profile.json（通过 `tools/profile_manager.py load`）。如果画像不存在，提示用户先建档。

### Step 2: 消息分类

将用户的消息归类为以下之一：
- 邀约 / 日常分享 / 关心 / 玩笑调侃 / 求助请教 / 深入话题 / 暧昧试探 / 回应TA

参考 `references/response_patterns.md` 中的分类定义。

### Step 3: 画像匹配

交叉引用画像层：

1. **Layer 2（沟通风格）**→ 预测回复的形式（长度、emoji、语气）
2. **Layer 3（情绪模式）**→ 预测回复的态度（消息是否触及热衷/回避话题）
3. **Layer 4（关系动态）**→ 预测回复的热情度（当前互动趋势、温度计读数）

### Step 4: 生成预测

生成 3-4 种回复变体，覆盖从积极到消极的完整谱系。

**每种变体必须包含**：

1. **预测回复内容**：用 crush 的说话风格写（参考 Layer 2 的 emoji、标点、消息长度习惯）
2. **概率估计**：基于画像置信度和互动趋势。所有概率之和 = 100%
3. **预计回复时间**：基于 Layer 2 的 response_time 分布
4. **热情度评分**：1-10 分
5. **潜台词分析**：这种回复背后可能的心理状态

**概率分配规则**：
- 画像置信度高（多数字段 `[observed]`）→ 最可能回复占 50-60%，分布集中
- 画像置信度低（多数字段 `[speculative]`/`[unknown]`）→ 最可能回复占 30-40%，分布分散
- 温度计 60°+ → 积极回复概率上调
- 温度计 40°- → 消极回复概率上调

### Step 5: 置信度与依据

给出整体预测的置信度：

```
置信度计算参考：
- 基础分 30%
- 有 ≥30 条聊天记录: +20%
- 有同类型消息的历史回复数据: +20%
- 画像 Layer 2 多数为 [observed]: +15%
- 画像 Layer 4 趋势明确: +10%
- 修正循环反馈过 ≥3 次: +5%
```

列出预测基于哪些数据，以及什么数据能提高置信度。

### Step 6: 消息优化建议

如果积极回复的概率 < 50%，主动提供一个优化版消息：
- 降低压力 / 提供台阶 / 借力打力 / 分步推进
- 给出优化后的概率估计
- 参考 `references/response_patterns.md` 的优化原则

⏱ 预计回复时间: {时间范围}
🌡 热情度: {N}/10

---

## 风格模拟规则

预测回复必须模拟 crush 的实际说话风格，不能写成"标准回复"。

- 如果 Layer 2 显示 TA 爱用 emoji → 回复中加入 TA 常用的 emoji
- 如果 TA 说话简短 → 回复保持简短
- 如果 TA 有口头禅/特定笑法 → 使用 TA 的表达方式
- 如果 Layer 2 数据为 `[unknown]` → 用中性风格写，但注明"风格可能不准"

## 特殊场景处理

### 数据严重不足

如果画像中 Layer 2 有 ≥50% 字段为 `[unknown]`：

```
⚠️ 数据不足提醒

目前画像信息太少，预测准确度有限。建议先：
1. 提供一些和 TA 的聊天记录（哪怕几十条）
2. 或者回答几个关于 TA 沟通习惯的问题

现在也可以预测，但置信度会很低（~30%），仅供参考。
要继续预测还是先补充信息？
```

### 高风险消息

如果用户的消息属于"暧昧试探"或"表白"类型，且温度计 < 50°：

```
⚠️ 风险提示

这条消息的暧昧程度较高，而目前温度计读数({N}°)还在友好区间。
发出后有较大概率收到尴尬/回避的回复。

建议考虑：
1. 先用更安全的方式试探（降低暧昧浓度版本："{建议}"）
2. 等温度再升一升（目前趋势{trend}，预计{时间}后到达暧昧区间）
3. 如果你觉得时机到了，就直接发——预测只是参考，你比我更了解你们的感觉

要继续预测原消息的回复吗？
```

### 用户情绪觉察

如果用户在短时间内反复修改同一条消息（≥3 次），温和提示：

> 看得出来你对这条消息挺纠结的。其实发消息没有"完美措辞"，TA 对你的态度不会因为一个用词就完全改变。放轻松，选一个你觉得最自然的版本就好。
