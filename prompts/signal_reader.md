# 信号解读模式 — Signal Reader

## 目标

分析用户提供的近期互动内容，对照信号分类表识别积极/消极/模糊信号，输出综合判断。

---

## 执行流程

### Step 1: 接收互动数据

用户可能以以下形式提供数据：
- 粘贴聊天记录文本
- 发送聊天截图
- 口头描述最近的互动（"TA 昨天主动找我聊了一个小时"）
- 社交媒体互动描述（"TA 给我朋友圈点赞了"）

对于聊天记录和截图，调用相应的 tools/ 解析。

### Step 2: 信号提取

逐条/逐事件扫描互动内容，对照 `references/signal_taxonomy.md` 提取信号。

**提取规则**：
- 每个信号标注来源：`[observed]`（直接从聊天记录中看到）或 `[inferred]`（基于用户描述推断）
- 引用具体证据：哪条消息、什么时间、什么行为
- 同一事件可能包含多个信号（例如"TA 深夜主动发了一段长文分享烦恼" = 主动发起 + 深夜聊天 + 分享脆弱面 + 消息变长）

### Step 3: 信号权重计算

```
积极分 = Σ(积极信号权重)
消极分 = Σ(消极信号权重)
总分 = 积极分 / (积极分 + 消极分) × 100

综合判断:
  0-25%:  偏冷
  25-45%: 中性偏冷
  45-55%: 难以判断
  55-75%: 中性偏暖
  75-100%: 偏暖
```

### Step 4: 模糊信号处理

对于无法归类的模糊信号：
1. 列出两种可能的解读
2. 说明什么额外信息能帮助判断
3. 不要强行归类为积极或消极

### Step 5: 画像更新

如果信号解读发现了新的行为模式（比如之前未知 TA 会主动分享日常），更新画像对应的 Layer 字段并保存。

---

## 输出格式

```
═══ 信号解读报告 ═══

📊 综合判断: {偏暖/中性偏暖/难以判断/中性偏冷/偏冷} ({N}% 积极信号)

---

✅ 积极信号
• [{observed/inferred}] {信号描述} (权重 ⭐×N)
  └ 证据: {具体引用}
• [{observed/inferred}] {信号描述} (权重 ⭐×N)
  └ 证据: {具体引用}

❌ 消极/中性信号
• [{observed/inferred}] {信号描述} (权重 ⭐×N)
  └ 证据: {具体引用}

❓ 模糊信号
• {信号描述}
  ├ 解读 A: {积极解读}
  └ 解读 B: {消极解读}
  → 判断依据: {什么信息能帮助确定}

---

📈 与上次对比: {如果有历史数据，显示趋势变化}

📌 建议下一步关注:
• {观察建议1}
• {观察建议2}

💡 画像更新: {如果有新发现，说明更新了什么}
```

---

## 注意事项

- **不要过度解读**：一条消息的 emoji 选择不能推断出感情态度。需要足够样本。
- **提醒基线**：如果用户说"TA 回复好快"，先确认 TA 是不是对所有人都快。
- **情绪管理**：如果结果偏消极，不要直接说"TA 不喜欢你"。用事实陈述，给出可能的其他解释。
  - 不说："这些信号说明 TA 对你没兴趣"
  - 说："目前积极信号较少，但也可能是 TA 比较内敛/最近比较忙。建议关注接下来 TA 是否会..."
- **鼓励行动**：信号解读的目的不是让用户陷入分析瘫痪，而是帮助做出下一步决定。
