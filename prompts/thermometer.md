# 暧昧温度计 — Thermometer

## 目标

量化当前关系热度为 0-100 的温度值，展示趋势、依据和误差范围，帮助用户直观理解暧昧进展。

---

## 温度区间定义

| 区间 | 名称 | 含义 |
|------|------|------|
| 0-20 | 冷漠 | 几乎无互动，或互动完全事务性 |
| 20-40 | 礼貌 | 有互动但纯礼貌/同事/同学关系 |
| 40-60 | 友好 | 比普通朋友亲近一点，但暧昧信号不明显 |
| 60-80 | 暧昧 | 明显超出普通朋友的互动，有暧昧信号 |
| 80-100 | 明确 | 几乎可以确定双方有意思，差一个挑明 |

---

## 温度计算方法

### 基础分

从 Layer 4（关系动态）提取：

| 指标 | 计算方式 |
|------|----------|
| 互动频率 | 每天聊 = 高基础分；每周 1-2 次 = 中等；更少 = 低 |
| 趋势 | increasing = +5；stable = +0；decreasing = -5 |
| 内部梗数量 | 每个 +2，上限 10 |
| 混合信号 | 每个 -3（不确定性本身是降温因素） |

### 信号分加权

读取最近一次信号解读报告（如有），或基于画像中的 Layer 4 数据：

```
温度 = 基础分 + Σ(积极信号 × 权重) - Σ(消极信号 × 权重)
归一化到 0-100 区间
```

### 误差范围

```
如果画像中 [observed] 比例 ≥ 70% → 误差 ±8°
如果 [observed] 比例 40-70% → 误差 ±12°
如果 [observed] 比例 < 40% → 误差 ±18°
如果数据源 ≤ 1 → 额外 +5° 误差
```

### 各区间建议模板

**0-20°（冷漠）**：
> 目前互动很少。可以先从轻量级互动开始（点赞、群聊回复），建立存在感。不建议直接发私人消息。

**20-40°（礼貌）**：
> 有基本互动但还在礼貌范畴。可以通过共同话题/共同朋友增加接触频率。关注 TA 是否会回应你的主动。

**40-60°（友好）**：
> 已经超过普通朋友了。现在是建立更深连接的好时机——分享更个人的话题、增加单独相处的机会。关注 TA 是否也在推进。

**60-80°（暧昧）**：
> 在暧昧区间了！TA 的很多行为已经超出"just friends"。可以开始更明确的试探。但注意：暧昧也可能只是 TA 的社交风格，关键看 TA 对你是否"特殊"。

**80-100°（明确）**：
> 信号已经很明显了。如果你也喜欢 TA，是时候考虑表明心意了。继续暧昧下去反而可能让对方困惑或失去耐心。

## 历史追踪

每次计算温度后，更新画像的 `layer_4_dynamics.thermometer` 字段：

```json
{
  "value": 68,
  "trend": "+4/week",
  "last_updated": "2026-04-03",
  "history": [
    {"date": "2026-03-20", "value": 52},
    {"date": "2026-03-27", "value": 58},
    {"date": "2026-04-03", "value": 68}
  ]
}
```

如果有历史数据，在输出中显示趋势线：

```

  3/27  ██████████████░░░░░░  58°

```

## 注意事项

- 温度计是**主观量化工具**，不是科学测量。始终强调这一点。
- 不要让用户把温度当成 KPI 来刷。温度的意义在于帮助理解趋势，不是追求数字。
- 温度下降不一定是坏事——可能只是互动频率的自然波动。趋势比单次读数重要。
