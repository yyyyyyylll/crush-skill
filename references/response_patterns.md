# 暧昧期常见回复模式 — Response Patterns

预测引擎的参考库。按消息类型分类 crush 的常见回复模式。

---

## 消息类型分类

用户发给 crush 的消息可归为以下类型：

| 类型 | 定义 | 示例 |
|------|------|------|
| **邀约** | 提议一起做某事 | "周末有空吗？""想不想去看那个展？" |
| **日常分享** | 分享自己的生活片段 | "今天吃了一家超好吃的拉面""加班到现在😭" |
| **关心** | 表达对 TA 的关注 | "你感冒好点了吗？""今天忙不忙？" |
| **玩笑/调侃** | 轻松的互动 | "你那个表情包太离谱了哈哈""又偷懒？" |
| **求助/请教** | 以请教为由搭讪 | "你推荐的那个APP叫什么来着？" |
| **深入话题** | 推进亲密度的话题 | "你小时候想当什么？""最近有什么烦心事吗？" |
| **暧昧试探** | 带有暧昧暗示的消息 | "你怎么这么可爱""要不要当我的专属咖啡师" |
| **回应 TA** | 回复 TA 发来的消息 | (取决于 TA 发了什么) |

---

## 各类型的回复模式谱

### 邀约类

```
热情接受 ───── 有条件接受 ───── 模糊回应 ───── 婉拒 ───── 冷拒
"好啊！几点？"  "看看时间"      "那个展还行"   "可能没空"  "不了"
```

**预测关键因素**：
- TA 之前接受过邀约吗？（Layer 4 数据）
- 邀约的压力大小（"一起吃饭" < "周末出去玩" < "看电影"）
- 是否给了 TA 台阶（"有空的话"比"周末去吧"压力小）
- 当前温度计读数

**回复时间模式**：
- 秒回/几分钟 → 大概率积极（没犹豫）
- 30分钟-1小时 → 可能在考虑/纠结
- 几小时 → 可能在想怎么拒绝又不伤感情
- 不回 → 通常不是好信号

### 日常分享类

```
积极回应 ────── 常规回应 ────── 敷衍回应 ────── 不回应
"好好吃！在哪？  "哈哈不错"       "嗯嗯"         (已读)
下次带我去！"
```

**预测关键因素**：
- TA 是否也会主动分享日常？（互惠性）
- 分享内容是否触及 TA 的兴趣点（Layer 3）
- 时间点（工作时间 vs 晚上放松时）

### 关心类

```
感动回应 ────── 正面接受 ────── 礼貌回应 ────── 回避
"你怎么这么贴心  "好多了谢谢～"   "还行"          (转移话题)
好多了！你呢？"
```

**预测关键因素**：
- TA 的 vulnerability_level（Layer 3）——自我保护意识强的人可能对关心持防备
- 关心的频率——第一次关心和第十次关心的回应不同
- 措辞是否自然——过度关心可能让人不适

### 玩笑/调侃类

```
接梗回击 ────── 开心回应 ────── 普通回应 ────── 不接茬
"你才偷懒！     "哈哈哈哈笑死"  "哈哈"          "嗯"
我可是加班狗好吧"
```

**预测关键因素**：
- TA 的 humor_style（Layer 3）——是否是能接梗的类型
- 你们之间的默契程度（Layer 4 inside_references）
- 调侃的分寸——太轻无感，太重可能冒犯

### 深入话题类

```
敞开聊 ──────── 浅聊 ──────── 带过 ──────── 回避
"说来话长…      "还好吧，你呢？" "嗯，都过去了"  (转移话题)
(长段文字)"
```

**预测关键因素**：
- TA 的 vulnerability_level 是预测这类消息最关键的指标
- 关系阶段——认识 2 周和 2 个月的回应不同
- 提问方式——开放式 > 封闭式

### 暧昧试探类

```
接受暧昧 ────── 害羞回应 ────── 装傻 ────── 明确划线
"你更可爱好吧～" "讨厌～"        "哈哈你搞笑"  "别闹"
```

**预测关键因素**：
- 当前温度计读数——低于 50° 时暧昧试探风险高
- TA 对暧昧的接受度——之前是否有过暧昧互动
- 措辞的模糊度——越模糊越安全（"你好有趣" vs "我好喜欢你"）

## 回复时间参考模型

| 场景 | 积极信号 | 中性 | 消极信号 |
|------|----------|------|----------|
| 日常聊天 | 5分钟内 | 10-30分钟 | 2小时+ |
| 邀约 | 10分钟内 | 30分钟-1小时 | 3小时+ |
| 暧昧试探 | 1-5分钟 | 15-30分钟 | 1小时+ |
| 深夜聊天 | 即时 | 5-15分钟 | (去睡了) |

**重要**：以上仅为参考基线。实际预测必须以 TA 的个人回复速度分布（Layer 2 response_time）为准。一个平时都 1 小时才回的人，30分钟回就已经是积极信号了。

---

## 消息优化原则

当预测结果不理想时，可建议用户调整消息：

1. **降低压力**：把确定性语气改为开放性（"去吗？" → "有兴趣吗？" → "你听说过吗？"）
2. **提供台阶**：给 TA 一个不丢面子的拒绝选项（"有空的话"、"不方便也没事"）
3. **借力打力**：借用共同话题/第三人降低刻意感（"小王说那家店不错" vs "我想带你去那家店"）
4. **分步推进**：把一个大邀约拆成小步骤（先聊展览 → 再问有没有兴趣 → 最后约时间）
5. **匹配风格**：消息风格尽量贴近 TA 的沟通风格（TA 用短句你也用短句）
